# AGENTS.md（仓库根规则）

## 作用域
本文件作用于仓库根目录及其所有子目录。
如果子目录存在更深层的 `AGENTS.md`，则子目录规则优先。

## 规则优先级
1. 用户当前指令
2. 更深层级 `AGENTS.md`
3. 当前文件（根 `AGENTS.md`）
4. `README.md` / `README_2.md` / 其他说明文档

> 说明：`README` 是说明文档；`AGENTS.md` 是执行规则文档。

## 第二阶段默认目标
- 以“多节点协同检索 + 中心聚合回答”为主线推进。
- 所有联调改动优先围绕可扩展、可维护、可审计。

## 强制交付项（每次新增功能）
每次新增功能或改动行为，都必须同步完成以下三项：

1. **功能归属声明**
   - 明确该功能由哪些文件负责（主责文件 + 协同文件）。
2. **扩展性审查**
   - 不是只看是否报错，必须评估后续扩展成本与边界稳定性。
3. **必要重构**
   - 如果发现重复逻辑、边界污染、硬编码扩散，必须当次重构，不允许留“以后再说”。

## 文档同步门禁
当改动影响模块职责时，必须同步更新：
- `docs/architecture/modules-index.md`
- `docs/architecture/extension-review-checklist.md`（追加当次审查记录）

## 渐进式上下文原则
- 根规则保持精简，只定义全局硬约束。
- 具体实现细则下沉到更深层 `AGENTS.md`，按目录逐级加载。
- 避免在根规则堆叠过多低层细节，减少无效上下文。

## GSAP 动画质量基线（全局）
- 首屏关键动画阶段，禁止并发初始化重负载模块（如大图谱/复杂物理布局）；重模块必须按需懒挂载。
- 动画属性优先 `transform` / `opacity`；避免高频动画中使用会放大重绘开销的属性（尤其 `filter`/`blur`/大面积阴影）。
- 禁止在 hover/频繁交互中全量重建时间轴；应仅更新“变更目标”（例如上一项与当前项）。
- SVG 动画优先轻量路径：能不用滤镜就不用滤镜，能用纯描边/透明度实现就不引入额外合成层。
- 性能验收至少覆盖用户常用环境（当前项目默认检查：Edge + 125% 缩放），并记录结论与残余风险。
